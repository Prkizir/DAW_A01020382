Cuestionario

¿Cual es la diferencia entre internet y la World Wide Web?

Internet es el medio por donde se interconectan diversos dispositivos a una red. La World Wide Web es la manera en que accedemos a la información en el medio a través de protocolos como HTTP (HyperText Transfer Protocol).

¿Cuál es el proposito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Es un método que se usa para solicitar información. Se debe evitar usar este método para mandar información.

HEAD: Este método se usa para obtener información sobre la entidad que implica la solicitud sin transferir la entidad como tal. Es bastante similar a GET, pero sin regresar lo que la entidad responde, puede servir para verificar qué se va a mandar antes de un GET.

POST: Se usa para mandar información al servidor.

PUT: Actualización de datos.

PATCH: Actualización de datos.

DELETE: Manda una solicitud al servidor para eliminar un recurso en específico

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usario y contraseña en algún sitio? ¿Por qué?

POST, ya que se maneja información sensible. Es mejor usar este método ya que la información no se puede observar en el URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Para acceder a una página a través de un navegador web, se utiliza el GET. Usa una referencia a la página que se trata de acceder, no se busca mandar información.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

200: OK; No ocurrió ningún error. El código 200 indica que la petición se completó con éxito.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

404: Not Found; Indica que no se encontró ningún recurso que coincida con la solicitud. Se usa también para limitar a los usuarios de acceder a cierto contenido sin la necesidad de dar una explicación del por qué ese recurso no está disponible para ciertos usuarios por lo que no es necesario corregir un sitio web con este error a menos que se busque un resultado diferente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

500: Internal Server Error; Este error indica que hubo un problema en el servidor que le impide completar la petición, por lo que es importante encontrar el problema y corregirlo.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando se habla de un atributo despreciado significa que el sistema lo sigue soportando pero que ya no se recomienda su uso. Algunos elementos despreciados son:

- BASEFONT: Cambiar fuente base (pref. style sheet)

-CENTER: Bloque centrado (pref. style sheet)

-FONT: Cambiar fuente (pref. style sheet)

-U: Texto subrayado (pref. style sheet)

-MENU: Lista (pref. UL)

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 tiene una sintaxis más simplificada y es más flexible en cuanto a funcionalidad. Se pueden integrar nuevos elementos multimedia como videos y audio directamente. Existe una comunicación full dúplex entre servidor y cliente. HTML4 al tener más tiempo, es compatible con casi todos los buscadores web.

¿Qué componentes de estructura y estilo tiene una tabla?

Los componentes que definen la estructura basica de una tabla son:

<tr>: indica las filas en la tabla

<th>: este tag define el header del segmento de la tabla

<td>: Los datos de la tabla se definen con este tag

Los estilos de la tabla pueden variar dependiendo las configuraciones que CSS provee.
