Cuestionario

¿Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

El !important debería ser utilizado como una solución de último recurso, ya que el mismo es radical. Es básicamente una regla que sobreescribe a las otras y hay que tener cuidado con qué elementos aplicamos esta regla, ya que aplicado en elementos con reglas de menor prioridad, puede sobreescribir cualquier especificación sobre dicho elemento y puede afectar el trabajo colaborativo.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Es importante analizar la imagen que se pueda utilizar de fondo ya que el tamaño sí importa ( ͡° ͜ʖ ͡°).
Existen varios atributos que pueden modificar una imagen de fondo y cosas inesperadas pueden suceder si no se manejan dichos atributos correctamente, además de que la imagen puede no ser un vector y la calidad de la misma puede verse altamente afectada y también puede que no cubra toda la pag. sin repetirse.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Personalmente recomendaría el uso del % ya que dichas son adaptables al dispositivo donde se esté visualizando el contenido. px y pt pueden tener definiciones diferentes entre dispositivos y al ser valores fijos, pueden afectar la estructura de la pag.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Porque al hacer una petición si se usa la versión minimizada, el ancho de banda que se usa para enviar la información contenida en el CSS es mucho menor, lo que reduce los tiempos de respuesta y por ende una carga más rápida.
